🚨 Real-Time Fraud Detection using Snowflake + XGBoost
📖 Project Overview

This project demonstrates an end-to-end Fraud Detection system built using:

Snowflake as the data warehouse

SQL-based feature engineering

XGBoost for machine learning

Python (Snowflake Connector) for ML integration

The goal is to simulate a production-style fraud detection pipeline that:

Ingests transaction data into Snowflake

Performs feature engineering inside Snowflake

Trains a machine learning model

Generates fraud probability predictions

Stores predictions back into Snowflake

This project was built as part of the AI/ML Engineer technical evaluation.

🧠 Business Problem

Online transaction systems suffer financial losses due to fraudulent activities.

Fraud detection systems must:

Identify high-risk transactions

Handle extreme class imbalance

Provide probability-based risk scoring

Integrate seamlessly with data warehouse infrastructure

This project simulates a real-world fraud detection pipeline similar to what payment platforms and e-commerce systems deploy.

Class Distribution

- Fraud: depends on generator (default setting in `data_generation/generate_data.py` is approximately 5%)

This simulates real-world fraud detection scenarios where fraud is rare. You can generate the sample CSV locally with the included script:

```
python data_generation/generate_data.py
```

The fraud rate is configurable in the generator and the actual distribution may vary depending on generation logic.

🔬 Feature Engineering (Inside Snowflake)

A new table FRAUD_FEATURES was created using SQL transformations.

Engineered Features
Feature	Purpose
HIGH_AMOUNT_FLAG	Flags large transactions
NEW_ACCOUNT_FLAG	Identifies recently created accounts
LOGIN_RISK_FLAG	Captures suspicious login attempts
VELOCITY_FLAG	Detects burst transaction activity
RISK_SCORE	Composite risk indicator

Feature engineering was performed inside Snowflake using SQL, demonstrating warehouse-native transformation capabilities.

🤖 Model Selection
Model Used: XGBoost Classifier

Why XGBoost?

Handles tabular data effectively

Strong performance on imbalanced datasets

Robust to non-linear relationships

Industry standard for fraud detection

Handling Class Imbalance

Fraud rate ≈ 0.5%, so we used:

scale_pos_weight = (# non-fraud / # fraud)

This prevents the model from predicting all transactions as non-fraud.

📈 Model Performance

ROC-AUC Score: 0.99996

Classification Report
Class	Precision	Recall	F1-Score
Non-Fraud (0)	1.00	1.00	1.00
Fraud (1)	0.99	0.96	0.98

Note: Performance is high due to synthetic data with rule-based fraud generation.

🔁 Prediction Pipeline

The trained model:

Loads features from Snowflake

Generates fraud probability (FRAUD_PROBABILITY)

Assigns risk category:

HIGH (> 0.8)

MEDIUM (0.4–0.8)

LOW (< 0.4)

Stores results in FRAUD_PREDICTIONS table

This simulates a production inference pipeline integrated with Snowflake.
