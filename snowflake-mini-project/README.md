# 🚨 Real-Time Fraud Detection — Snowflake + XGBoost

## Overview

This repository demonstrates an end-to-end fraud detection pipeline using Snowflake for data warehousing and XGBoost for machine learning. It includes a synthetic dataset generator, SQL-based feature engineering (assumed to run in Snowflake), model training, and an inference script that writes predictions back to Snowflake.

## Architecture

Synthetic CSV (generated by script)
↓ 
Snowflake (FRAUD_DATA)
↓
SQL Feature Engineering (FRAUD_FEATURES)
↓
Python + XGBoost Model Training
↓
Model Saved (`fraud_model.pkl`)
↓
Prediction Pipeline → Snowflake (FRAUD_PREDICTIONS)

## Dataset & Generation

Synthetic transaction data is produced by the included generator at `data_generation/generate_data.py`.

Generate the CSV locally with:

```bash
python data_generation/generate_data.py
```

Default generator settings:
- `NUM_ROWS = 100000`
- `FRAUD_RATIO = 0.05` (configurable in the script)

The generator uses a simple rule-based fraud signal which may result in different class distributions depending on parameter settings.

## Files

- `data_generation/generate_data.py` — synthetic dataset generator (writes `fraud_data.csv`).
- `snowpark/train_model.py` — loads `FRAUD_FEATURES` from Snowflake, trains XGBoost, saves `fraud_model.pkl`.
- `snowpark/predict.py` — loads features from Snowflake and `fraud_model.pkl`, computes probabilities, writes `FRAUD_PREDICTIONS` back to Snowflake.
- `.env` — local Snowflake credentials (excluded from VCS via `.gitignore`).
- `fraud_data.csv` — generated dataset (not committed by default).

Core dataset columns:

- `TRANSACTION_ID`, `USER_ID`, `TRANSACTION_AMOUNT`, `ACCOUNT_AGE_DAYS`,
- `FAILED_LOGINS`, `DEVICE_CHANGE`, `COUNTRY_RISK_SCORE`, `TRANSACTIONS_LAST_1H`, `IS_FRAUD`

Class distribution depends on the generator configuration. The included generator default sets fraud ~5%.

## Setup

1. Create a Python virtual environment and install dependencies.

```bash
python -m venv .venv
# Windows PowerShell
.\.venv\Scripts\Activate.ps1
# or cmd
.\.venv\Scripts\activate.bat
pip install -r requirements.txt
```

If `requirements.txt` is not present, install the basics:

```bash
pip install pandas xgboost scikit-learn joblib python-dotenv snowflake-connector-python
```

2. Add your Snowflake credentials to a `.env` file at the repo root using the keys: `SNOWFLAKE_USER`, `SNOWFLAKE_PASSWORD`, `SNOWFLAKE_ACCOUNT`, `SNOWFLAKE_WAREHOUSE`.

## Usage

1. Generate synthetic data (optional local CSV):

```bash
python data_generation/generate_data.py
```

2. Train the model (assumes `FRAUD_FEATURES` table exists in Snowflake):

```bash
python snowpark/train_model.py
```

3. Generate predictions and write to Snowflake:

```bash
python snowpark/predict.py
```

## Modeling Notes

- Model: `XGBoost` classifier (saved as `fraud_model.pkl`).
- Imbalance handling: training sets `scale_pos_weight = #neg / #pos`.
- Metrics: the synthetic setup often yields very high ROC-AUC due to rule-based labels — treat scores as illustrative.
